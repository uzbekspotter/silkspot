# Подключение Supabase к SkyGrid Pro

## Зачем это нужно
Supabase — это база данных в облаке. Без него сайт работает на демо-данных.
После подключения:
- Реальный логин (Google / email)
- Загруженные фото сохраняются
- База самолётов наполняется пользователями
- При вводе регистрации — данные из нашей БД, не только hexdb

---

## Пошаговая инструкция

### 1. Создать аккаунт
Идёшь на https://supabase.com → Sign Up (бесплатно)
Бесплатный план: 500 MB базы, 1 GB хранилища — для старта хватит.

### 2. Создать проект
- New Project → называешь "skygrid-pro"
- Выбираешь регион: **Frankfurt (eu-central-1)** — ближайший к Ташкенту
- Придумываешь Database Password (сохрани его!)
- Ждёшь ~2 минуты пока создаётся

### 3. Запустить миграции (создать таблицы)
В Supabase: левое меню → **SQL Editor** → New query

Запускаешь по очереди (копируешь содержимое и жмёшь RUN):
1. `supabase/migrations/001_schema.sql`
2. `supabase/migrations/002_rls.sql`
3. `supabase/migrations/003_seed.sql`
4. `supabase/migrations/004_aircraft_details.sql`

### 4. Получить ключи
В Supabase: левое меню → **Settings** → **API**

Копируешь два значения:
- **Project URL** — выглядит как `https://abcdefgh.supabase.co`
- **anon public** key — длинная строка начинается с `eyJ...`

### 5. Создать .env файл
В папке `C:\WWW\skygrid-pro\` создай файл `.env`
(просто скопируй `.env.example` и переименуй в `.env`)

Вставь свои ключи:
```
VITE_SUPABASE_URL=https://abcdefgh.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5c...
VITE_APP_URL=http://localhost:3000
```

### 6. Перезапустить сервер
```
Ctrl+C  (остановить)
npm run dev  (запустить снова)
```

## Что происходит после подключения

### Lookup самолётов (Upload страница)
```
Юзер вводит: UK32101
      ↓
1. Запрос в Supabase → aircraft таблица
   Нашли? → возвращает ВСЁ: тип, оператор, MSN, first flight, config, engines
      ↓
2. Не нашли в Supabase → запрос в hexdb.io (как сейчас)
   → возвращает: тип + оператор (без MSN/config/engines)
      ↓
3. Не нашли нигде → юзер вводит вручную
```

### Краудсорсинг — как база наполняется
Каждый раз когда юзер заполняет MSN / Config / Engines при загрузке фото
и нажимает Submit — эти данные сохраняются в Supabase.

Следующий юзер который загружает фото того же самолёта —
получает эти данные автоматически.

Так работает Planespotters.net — тысячи людей дополняют базу.

### Верификация данных
- Новые данные от юзеров: `is_verified = false` (принимаются, но помечены)
- Проверенные модератором: `is_verified = true` (не могут быть изменены юзерами)
- В Admin Panel: раздел "Aircraft Data Review" для проверки
